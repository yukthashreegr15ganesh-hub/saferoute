SafeRoute
Comprehensive Feature Manual & Expansion Guide

This document outlines every existing feature within the SafeRoute application as of the current build. It is designed to help you review current capabilities and brainstorm new features to add next.

Tip: You can press Ctrl + P (or Cmd + P) in your browser right now and select "Save as PDF" to keep a permanent copy of this manual!

1. Intelligent Routing & Navigation

Current Implementation

  • Safety-First Pathfinding: Unlike Google Maps which optimizes for time, SafeRoute generates three route options: SAFE, MODERATE, and DANGER.
  • OSRM Integration: Connects to Open Source Routing Machine to draw actual turn-by-turn geometry.
  • Dynamic Safety Tags: Routes are automatically tagged with metrics like "Well lit ✓", "CCTV ✓", "Isolated Area ✗", or "Active Crowd".
  • Voice Navigation: An integrated UI overlay provides step-by-step turn guidance.

Ideas for New Features to Add:

  • Live Apple/Google Maps Redirect: Allow users to export the "Safe" route directly to Google Maps or Apple Maps for native carplay/Android Auto navigation.
  • Deviation Alerts: If the GPS detects the user has drifted more than 50 meters off the "SAFE" path, automatically trigger an Aria check-in or vibrate the phone.

2. Guardian SOS System

Current Implementation

  • Multi-Trigger Button: The SOS button responds differently based on how you interact with it:
    • Single Tap: Starts a 3-second cancelable countdown.
    • Double Tap: Instant activation (no countdown).
    • Long Hold (3s): "Stealth Mode" activation (vibrates instead of flashing screen).
  • SMS Broadcasting: Integrates with Twilio API to automatically send SMS messages to up to 5 emergency contacts with a live Google Maps link of the user's coordinates.
  • Low Battery Alert: Automatically broadcasts the last known location to contacts if the phone battery drops below 5%.

Ideas for New Features to Add:

  • Fake Phone Call Simulator: A button that triggers a realistic ringing phone screen and plays a pre-recorded voice on the other end to deter harassment.
  • Hardware Trigger: Integrate with the phone's volume buttons (e.g., clicking volume-down 5 times rapidly triggers SOS even if the phone is locked).

3. ARIA (AI Co-Pilot)

Current Implementation

  • Anthropic Claude Integration: A floating chat assistant capable of answering safety questions contextually.
  • Whisper Mode: A toggle that mutes all app audio and uses haptic feedback (vibrations) so the user can communicate with the AI silently.
  • Scheduled Check-ins: Users can say "Check on me in 30 minutes". If the user doesn't respond to the prompt 30 minutes later, the app automatically triggers the SOS protocol.

Ideas for New Features to Add:

  • Voice Activation: Implement a wake word (e.g., "Hey Aria, I need help") using the Web Speech API to trigger features hands-free.
  • Camera Scene Analysis: Allow users to snap a photo of a dark street and have Aria analyze it for potential hiding spots or safety risks using a Vision API.

4. Community & Sentinel Network

Current Implementation

  • Shadow Walk: A peer-to-peer tracking feature where a trusted contact can watch your live dot move on their map until you reach your destination.
  • Incident Reporting: Users can drop pins for "Harassment", "Poor Lighting", or "Isolated Areas". These show up as heatmaps for other users.
  • Safe Spots Directory: Dynamically highlights nearby 24/7 hospitals, police stations, and late-night shops based on live coordinates.

Ideas for New Features to Add:

  • Verified Guardian Program: Allow volunteers (e.g., shop owners, security guards) to register as "Verified Guardians" on the map, providing safe havens.
  • Crowd-Sourced Lighting Scores: Gamify the app by letting users rate the street lighting of their current block by tapping a thumb up/down, rewarding them with "Safety Points".

5. Core App Engine & User Profile

Current Implementation

  • Onboarding Flow: A secure PIN setup and emergency contact selection on first launch.
  • Night Ops Mode: A specialized high-contrast dark theme optimized to prevent screen glare from blinding the user at night.
  • App Disguise (Placeholder): Groundwork exists for a stealth UI.

Ideas for New Features to Add:

  • Decoy PIN / App Disguise: If an attacker forces the user to open the app, entering a "Decoy PIN" opens a fake calculator or generic news app while secretly dispatching police/SOS.
  • Offline Maps Mode: Download city data so routing and safe-spot finding works even if cellular data is jammed or out of range.
